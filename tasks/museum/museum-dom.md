## museum-dom

**Ваша задача** – "оживить" ранее сверстанную landing page Лувра – одного из крупнейших и наиболее популярных художественных музеев мира.

## Ключевые навыки:

- легкоподдерживаемый читаемый код
- работа с различными слайдерами
- написание кастомного видеоплеера
- реализация сравнения изображений
- анимация изображений при прокрутке
- калькулятор продажи товаров
- подключение интерактивной карты

## Работа над проектом:

Вам даётся 2 недели на оживление ранее написанной верстки, используя современные возможности js

## Технические требования

1. легкоподдерживаемый читаемый качественный код, при написании кода следуйте гайдлайнам:
   - https://javascript.info/coding-style
   - https://developer.mozilla.org/en-US/docs/MDN/Guidelines/Code_guidelines/JavaScript
2. приложение корректно отображается и работает в браузере Google Chrome последней версии
3. Разрешается использование библиотек для написания слайдеров, но рекомендуется написать слайдеры вручную
4. Разрешается использование mapbox
5. Запрещено использование React, Angular, Vue, Svelte, других фреймворков и библиотек

## Критерии оценки

**Максимальный балл за задание – 210 баллов**

### Слайдер в секции Welcome (40 max)

- при клике на любой из квадратов соответственно меняется изображение +3
- при клике на любой из квадратов меняется номер картины +2
- при клике на любой из квадратов он становится активным +3
- при клике на любую из стрелок меняется картина +3
- при клике на любую из стрелок меняется номер картины +2
- при клике на любую из стрелок соответственно меняется активный квадрат +3
- при клике на левую стрелку на первой картинке произойдет смена на последнюю картинку, при клике на правую на последней картинке произойдет смена на первую +3
- при "захвате" картины мышкой картина следует за мышкой по горизонтальной оси +2
- если "протащить" картину мышкой влево или вправо и отпустить, она сменится на картинку справа при свайпе влево и наоборот +3
- если при отпускании курсор не вышел за границы картины, а расстояние перетаскивания очень мало (<10px), то картина вернется в исходное состояние и не изменится. Если же больше 10px, то картина свайпнется +2
- если при перетаскивании курсор уходит за границы картины, она перелистывается в сторону курсора +2
- при свайпе картин меняется номер активной картины +2
- при свайпе картин меняется активный квадрат +3
- нет ситуации, когда картина находится не по центру при отпущенной клавише мыши или видны одновременно две картины +5
- сдвиг при всех способах смены картины происходит плавно +2

### Секция Video (40 max)

Слайдер и его взаимодействие с основным видео:

- При клике на любую из стрелок соответственно меняются видео в слайдере +3
- После клика на любую из стрелок главное видео совпадает с первым видимым видео из слайдера +3
- При клике вправо на последней точке или влево на первой происходит перематывание на первую или последнюю точку соответственно (речь о видео в слайдере и главном видео) +3
- При клике на любую из стрелок меняется активная точка +3
- При клике на любую из точек соответственно меняются видео в слайдере +3
- При клике на любую из точек меняется главное видео, оно совпадает с первым видимым в слайдере +3
- Видео в слайдере меняются плавно +3

Кастомный видеоплеер:

- При клике на само видео происходит toggle статуса проигрывания (включается, если было на паузе и ставится на паузу, если было включено), при этом исчезает или появляется знак проигрывания на видео и изменяется знак проигрывания или паузы слева в контроллерах видео +4
- При проигрывании видео движется длинный ползунок +2
- При движении длинного ползунка вручную видео запускается, если было на паузе, и меняется момент в самом видео +3
- При клике на иконку динамика происходит toggle звука и самой иконки (включается или выключается, соответственно изменяется иконка) +3
- При движении ползунка звука изменяется громкость видео +2
- Если ползунок звука вручную опустить до 0, то и иконка динамика также станет зачеркнута. Аналогично если вручную поднять громкость при выключенном динамике +3
- При нажатии на кнопку полноэкранного режима видео в него переходит. При нажатии повторно видео из него выходит. Нажатие на клавишу для выхода из полноэкранного режима `Esc` не проверяем и не оцениваем +2

**Взаимодействие плеера с клавиатурой (в том числе и клавишей `Esc`) не оценивается, но рекомендуется к выполнению**

**Итого за эту секцию максимально N баллов**

### Секция Explore (30 max)

В этой секции проверяем ползунок и смену фильтра на картинке

- Ползунок никогда не выходит за границы картины +15
- Ползунок корректно работает и меняет фильтр одной из частей картины +15

### Секция Gallery (30 max)

- При прокрутке вниз и появлении картин они плавно появляются, увеличиваясь и создавая эффект выплывания +18
- При повторном поднятии вверх и прокрутке вниз с появлением картин анимация повторяется +6
- При прокрутке вверх ничего не происходит +6

### Секция Tickets (25 max)

- При изменении типа билета меняется цена за билет Basic и Senior +5
- Для каждого типа билета своя цена (20, 25, 40 для Basic и в два раза меньше для Senior) +5
- При изменении количества билетов пересчитывается общая цена +5
- При изменении типа билета пересчитывается общая цена +10

### Tickets модальное окно (25 max)

- В entry ticket указана цена Basic, Senior и она не равна null или undefined +10
- При увеличении кол-ва билетов пересчитывается total +5
- При увеличении кол-ва билетов меняется их количество в overview +5
- При увеличении кол-ва билетов меняется суммарная цена для Basic и Senior +5

### Секция Contacts (30 max)

- Карта подключена +10
- Маркеры отображаются +5
- Карта интерактивна +10
- Карта выглядит так, как на макете (стиль) +5

Разница между максимальной оценкой за приложение (220 баллов) и максимально возможным количеством баллов за выполнение всех пунктов требований (210 баллов) позволит сгладить возможные ошибки проверяющих в ходе кросс-чека, неточности в описании задания, разное понимание требований задания проверяющим и проверяемым.
